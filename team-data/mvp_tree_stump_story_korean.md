# MVP 사양서: 오래된 그루터기의 이야기

## 제품 개요
- **제목:** 오래된 그루터기의 이야기
- **목표:** 오래된 나무 그루터기의 이야기와 교훈을 전하는 인터랙티브 디지털 체험을 제공하여 사용자들에게 감정적, 지적 몰입감을 선사

## 제품 오너/관리 관점
- **타겟 사용자:** 자연과 스토리텔링, 삶의 교훈에 관심 있는 어린이, 교육자, 가족 단위
- **핵심 가치:** 공감, 환경 인식, 나이듦과 기억의 아름다움
- **주요 기능:**
  - 인터랙티브 스토리텔링 (텍스트, 오디오, 비주얼)
  - 스토리 흐름에 영향을 주는 사용자 선택
  - 간단하고 접근 가능한 UI/UX
  - 진행 상황 추적 또는 북마크
- **성공 지표:**
  - 사용자 참여도 (체류 시간, 선택 횟수)
  - 사용자/교육자들의 긍정적 피드백
  - 재사용 또는 공유율

## 기술적 관점
- **플랫폼:** 웹 애플리케이션 (데스크톱/태블릿/모바일 반응형)

- **머신러닝 & NLP 통합:**
  - 개인화 스토리텔링: NLP를 사용하여 사용자의 독해 수준, 관심사, 감정 상태에 맞게 스토리 텍스트와 내레이션 적응
  - 인터랙티브 대화: 간단한 대화형 AI 구현으로 사용자가 그루터기에 질문하면 맥락을 이해한 스토리 기반 응답 제공
  - 감정 분석: 사용자 입력 (텍스트 또는 음성) 분석하여 스토리 톤 조정 또는 사용자 기분에 맞는 분기 제안
  - 콘텐츠 생성: 생성형 AI를 사용하여 사용자 선택과 피드백을 바탕으로 스토리 분기 확장 또는 새로운 결말 생성

- **기술 스택:**
  - 프론트엔드: React.js (또는 유사)
  - 백엔드: Node.js/Express (사용자 데이터용, 선택사항)
  - 데이터: 스토리 콘텐츠 및 사용자 진행 상황용 JSON 또는 간단한 DB
  - 머신러닝/NLP: Python (Hugging Face Transformers, spaCy, 또는 OpenAI API NLP 작업용)

- **MVP 범위:**
  - **위치 인식 랜딩 페이지**: 제주 설화 데이터베이스에서 GPS 기반 스토리 선택
  - **설화 기반 스토리 플로우**: 실제 전설 기반 2-3개 인터랙티브 선택
    - 예시: 영천악 용 전설 → 효돈천 신선 → 쇠소깍 사랑 이야기 연결
  - **문맥적 오디오 내레이션**: 문화적 발음이 포함된 제주 방언 옵션
  - **미니멀 디자인**: 제주의 화산 경관을 반영한 자연 영감 UI
  - **지능형 설화 Q&A**: 
    - "이 나무가 왜 여기 있어요?" → 지역 전설 기반 답변
    - "용이 정말 살았나요?" → 문화적 맥락 + 상상력 있는 응답
    - 위치별 설화를 위한 GPS 좌표 통합

- **향후 확장:**
  - 더 많은 스토리 분기
  - 사용자 계정 및 진행 상황 저장
  - 풍부한 미디어 (애니메이션, 사운드 효과)
  - 더 깊은 상호작용을 위한 고급 대화형 AI
  - 사용자 감정과 선호도를 활용한 적응형 스토리텔링
  - 음성-텍스트 및 텍스트-음성 변환을 활용한 음성 기반 상호작용과 내레이션

## 팀 통합 및 역할 분담

### 5인 팀 구조:
1. **김지현 (심리학/UX)**: 
   - 설화 기반 사용자 경험의 리서치 및 프로토타이핑
   - 연령별 적합한 스토리텔링을 위한 아동 심리학 연구
   - 위치 기반 내러티브와의 사용자 상호작용 프로토타이핑
   
2. **박소영 (토목공학/오디오)**: 
   - 설화 매핑을 위한 디자인 UX 및 데이터 전처리
   - GIS 기반 위치 분석 (ArcGIS/QGIS, Google Earth)
   - GPS 기반 스토리텔링을 위한 공간 데이터 통합
   - Booklet 디자인
3. **윤세정 (사회복지/창작)**: 
   - 제품 오너 및 프로젝트 관리
   - 문서화 및 이해관계자 커뮤니케이션
   - 콘텐츠 전략 및 설화 통합 감독
   - Booklet Gamification 
4. **이병남 (경영학/PM)**: 
   - 데이터 파이프라인 아키텍처 및 LangChain 통합
   - 제품 패키징 및 DevOps 구현
   - ML/NLP 인프라 및 배포 자동화
5. **정광식 (경영학/프론트엔드)**: 
   - 코딩 및 LLM 설계
   - 프론트엔드 개발 및 AI 통합
   - 모바일 최적화 및 대화형 AI 구현

윤세정정정, ㅇ이입이이벼병병병ㄴ나남남남- **1단계 (1주차)**: 연구 및 기획 (김지현, 윤세정)
- **2단계 (2주차)**: 핵심 개발 (정광식, 박소정, 이병남)
- **3단계 (2주차)**: 콘텐츠 및 게임 기능 (윤세정, 김지현)
- **4단계 (3주차)**: 테스팅 및 배포 (전체 팀)

### 제주도 통합:
- **타겟 위치**: 오름/둘레길 하이킹 트레일 (하례리 모델 적용)
- **설화 연구 방법론**: 
  - GPS 좌표 중심 반경 5km 내 설화 매핑
  - 기존 전설 통합 (영천악 용 전설, 쇠소깍 사랑 이야기, 효돈천 신선 설화)
  - 여러 지역 전설을 결합한 합성 스토리 창작
- **위치별 구현**:
  - 각 그루터기 위치를 특정 제주 설화에 매핑
  - 원본 전설 + AI 생성 연결 내러티브
  - 실제 장소명과 전통을 참조하는 인터랙티브 대화
- **문화적 요소**: 
  - 진정성 있는 체험을 위한 제주 방언 통합
  - 전통 스토리텔링 형식 (설화 구조)
  - 스토리 앵커로서의 지역 지형적 특징
- **관광 향상**: 자연 투어와 문화유산을 연결하는 디지털 동반자

## 게임 및 보상 시스템 구현

### 모바일 카메라 통합 (하례리 모델 구현):
- **설화 위치 인식**: 카메라가 특정 위치 식별 (영천악, 효돈천, 쇠소깍)
- **스토리별 사진 퀘스트**: 
  - "영천악에서 용의 모습을 찾아보세요" (찾기 게임)
  - "효돈천에서 신선이 앉았을 법한 바위를 찍어보세요"
  - "쇠소깍의 장군바위를 찾아 사랑 이야기를 완성하세요"
- **AR 설화 오버레이**: 
  - 스토리 순간 중 영천악 위에 나타나는 가상 용
  - 역사적 인물의 전통 한복
  - 인터랙티브 한라산 정령 가이드
- **GPS 기반 스토리 진행**: 위치 간 이동이 새로운 설화 챕터 트리거

### 카드 기반 발견 게임 시스템:
**제주 설화 발견을 위한 솔로 탐험 RPG 메커니즘 적용:**

**발견 메커니즘:**
- **새로운 스토리 요소 찾기**: 주사위를 굴려 발견할 설화 요소 수 결정
- **계시 방법**: 각 스토리 요소를 만나는 방법을 주사위로 결정:
  - 1-2: **힘든 발견** - 어려운 하이킹이나 등반 필요
  - 3-4: **갑작스런 만남** - 스토리 요소가 예상치 못하게 나타남
  - 5-6: **휴식 중 관찰** - 휴식을 취하는 중 발견

**스토리 요소 카테고리 (카드 무늬):**
- **♦ 다이아몬드 - 살아있는 설화**: 전설의 생물, 정령, 이야기 속 인물
- **♣ 클럽 - 성스러운 자연**: 고대 나무, 신비로운 식물, 자연 신전
- **♥ 하트 - 문화유산**: 돌 기념물, 폐허, 의식 장소
- **♠ 스페이드 - 자연 현상**: 화산 형성물, 샘, 동굴, 암석 형성물

**위치 맥락 (카드 숫자):**
- **A**: 키 큰 억새밭에서
- **2**: 한라산 달빛 아래에서
- **3**: 산 개울가에서
- **4**: 깊은 화산 협곡에서
- **5**: 오름(화산 원추) 정상에서
- **6**: 눈 덮인 봉우리에서
- **7**: 화산 분출구 근처에서
- **8**: 해안 절벽에서
- **9**: 용암 동굴에서
- **10**: 바위 돌출부에서
- **J**: 바람이 센 초원에서
- **Q**: 바다 해변에서
- **K**: 높은 전망대에서

**게임플레이 통합:**
- 플레이어가 제주 지역을 탐험하면서 디지털 "카드" 뽑기
- 각 카드 조합이 독특한 설화 발견 생성
- 앱에 기록된 일지 항목이 개인 스토리 컬렉션 생성
- 완료된 "행성"(위치)이 새로운 제주 지역 탐험 잠금 해제

### 디지털 배지 시스템 (문화유산 및 미션완수 중심):
- **설화 마스터**: 모든 설화 기반 스토리 경로 완성
- **제주 언어 학습자**: 제주 방언 발음 성취 잠금 해제
- **문화 가교 건설자**: 여러 지역 전설 성공적 연결
- **성지 방문자**: 스토리 내러티브 내 모든 설화 위치 방문
- **스토리텔러**: 전통 설화에 개인적 결말 창작 및 공유
